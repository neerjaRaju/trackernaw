# BRD-001 — Location Tracking Solution (Executive Summary)

**Version:** 0.2
**Date:** 2025-09-07
**Audience:** Business leaders, operations, finance, HR, compliance

---

## 1. Overview

A plug-in, embeddable Android SDK and cloud dashboard that provides live and historical location tracking for field employees. It supports offline capture & reliable sync, RBAC + SSO, and actionable analytics (distance, stops, heatmaps). It is designed for India BFSI compliance (DPDP, RBI vendor guidance) and multi-tenant resale.

## 2. Who is this for?

- **GFGC:** Property valuation, Field Investigation, Sales, SRO visits, HR institution visits (Case/Visit workflows).
- **Debt Collection** (Credgenics-like workflows): Agent route visibility, visit verification, manager oversight.
- **Any field-force business** needing mileage, productivity analytics, or compliance evidence.

## 3. Business Problems We Solve

- No single source of truth for where/when work happened.
- Manual or error-prone distance reporting; disputes in payroll/incentives.
- Limited visibility into agent availability, coverage, and stoppages.
- Fragmented tooling that doesn't embed into existing apps.

## 4. Value Proposition

- **Embeddable:** drop-in SDK; keep your app & UX.
- **Trustworthy data:** anti-spoofing, attestation, audit trails.
- **Real-time ops:** live map, staleness alerts, geofences.
- **Lower TCO:** streaming architecture, S3 cold storage, configurable sampling.
- **BFSI-ready:** consent flows, retention (90d raw, 13m aggregated), DR, access governance.

## 5. Scope by Phase

### Phase 1 — MVP (this release)

- Live map of active/idle users, presence health.
- Daily distance traveled and daily number of work (count of completed task check-ins).
- Route & stoppage analysis (auto-flag stops ≥ X min).
- Case/Visit form capture of lat/lon/time with `ref_id` and `type`.
- Historical views (90 days), heatmap, CSV daily export.
- SSO (Azure, Salesforce), RBAC, multi-tenant, DPDP controls.

### Phase 2 — Revenue Enablers

- Reimbursement/mileage engine (bike/car rate tables, exclusions).
- Odometer capture (photo + OCR) and reconciliation.
- Visit verifications (photo/OTP), richer geofence policies, advanced anti-spoofing.

### Phase 3 — Productivity & Intelligence

- iOS SDK, anomaly detection, schedule-based auto tracking, SLA/compliance dashboards, connectors to DWH/BI.

## 6. Success Metrics (Business KPIs)

- **Ops visibility:** ≥95% of on-shift agents visible live at p95 latency ≤5s.
- **Data trust:** <1% disputes on daily distance vs policy.
- **Adoption:** ≥90% SDK enabled on target field devices within 60 days of rollout.
- **Cost:** Storage/compute spend within budget per active user.

## 7. Customer Experience (Happy Path)

- Manager views live map; filters by team/module (Case/Visit); stale devices flagged.
- Agent starts tracking (auto or manual). Visits/cases submitted; lat/lon/time auto-attached.
- System computes distance and stoppages; end-of-day report lists distance and tasks completed.
- Execs see trends (heatmaps, coverage) and export CSV for payroll.

## 8. Compliance & Risk

- **DPDP:** consent, purpose, access/erasure requests; minimal data; retention enforced.
- **RBI (vendors):** DR, monitoring, access review, VAPT, SOC integration.
- **Risks:** battery impact (mitigate with adaptive sampling), OEM background limits (foreground service), location spoofing (attestation + heuristics).

## 9. Packaging & Commercial (Indicative)

- SaaS per active user / month tiers; overage for map-matching/API calls.
- Optional add-ons (Phase 2): reimbursement engine, odometer OCR, advanced anti-spoofing.

## 10. Approvals & Next Steps

- Approve Phase-1 scope & timelines.
- Nominate pilot tenants: GFGC (valuation/visits) and Collections.
- Kick off implementation per **SPEC-001**.

# BRD-001 v0.2 — Appendix & Phase-1 Shipping Sheet

**Date:** 2025-09-07

## A. What ships in Phase-1 (MVP)

- Android SDK embedded in host apps (Case/Visit), with offline capture, batching, start/stop, task check-ins, break tagging.
- Live map (≤5s) with presence, route view, stops ≥X min, daily distance, task count.
- Heatmap and CSV daily export.
- Geofences (basic): polygon/circle; entry/exit alerts (in-app/email/SMS).
- RBAC + SSO (Azure AD / Salesforce via Cognito) and DPDP/RBI controls.

## B. Platform Choices

| Layer | Technology |
|-------|------------|
| Gateway | Kong (REST + WebSocket) |
| Streaming | Kafka (MSK) with Flink jobs (Cleaner, Distance/Stops, Presence, Geofence) |
| Cache | Redis (ElastiCache) for presence |
| Database | RDS PostgreSQL (Multi-AZ) + PostGIS; S3 (raw/parquet); KMS encryption |

## C. Core API Endpoints (Cheat Sheet)

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/v1/locations` | Batch locations/events (≤50 points; ≤256KB) → `{ack_seq}` |
| GET | `/v1/config` | Remote policy JSON (ETag) |
| WS | `/v1/ws/presence` | Subscribe; `presence.update` events |
| POST | `/v1/tasks` | Create task |
| PATCH | `/v1/tasks/{external_task_id}` | Update task |
| GET | `/v1/presence` | Current presence per user |
| GET | `/v1/users/{id}/route?day&mapmatch` | User route for a day |
| GET | `/v1/daily-summaries` | Aggregated daily metrics |
| GET | `/v1/heatmap` | Heatmap data |
| GET | `/v1/reports/daily.csv` | CSV export for payroll |
| — | Geofence CRUD & events | Polygon/circle fences + entry/exit events |
| — | `/v1/teams`, `/v1/users`, `/v1/health` | Admin / health |

## D. KPIs & SLOs

- **Latency:** E2E ≤5s (p95) live updates
- **Accuracy:** ≤5m typical; cleaner + optional map-match
- **Reliability:** 99.9% SDK capture & backend

## E. Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Battery impact | Adaptive sampling; foreground service |
| OEM background limits | FGS + user education |
| Spoofing | **Real-GPS-only enforced:** Android SDK drops any fix flagged `Location.isMock` / `isFromMockProvider` at the source; Flutter app drops `Position.isMocked`; backend rejects `isMock` pings with `422 MOCK_LOCATION`. Backed by Play Integrity API + heuristics. |
